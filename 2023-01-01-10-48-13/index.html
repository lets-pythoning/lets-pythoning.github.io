笔记格式规范
#规范 2023/01/01

关于文件命名

  1. 如果是笔记文件, 命名长度不得超过 8 个字符, 若题目较长, 请在 H1 标题内写明, 而文件名应尽量缩短, 以言简意赅为优
  2. 如果是源文件, 如代码或图片等, 需放入单独的文件夹, 并使用英文命名, 空格用下划线代替, 长度无限制, 如果后面加编号的话, 不要使用单独的下划线分割, 也不要加括号, 直接附着在文件名后即可

关于文件夹命名

与文件命名相同, 英文需要用复数

关于笔记内容

  1. 数字与中文和英文与中文之间需要加空格, 要注意行内公式与正文之间不需要空格
  2. 除一级标题以外, 每个标题前后有且仅有一个回车
  3. 每个公式块和引用前后均不加回车, 其他块状结构前后都加回车, 列表每项之间不加回车, 列表前后加回车, 纯文本两段之间加回车 ( 即空出一行 )
  4. 所有标点使用英文, 任意英文括号两侧均有一个空格, 逗号和句号这样的分割型标点及冒号后有一个空格, 引号外加空格, 引号内不加空格
  5. 对于空格和回车: 全文任何地方不会出现连用, 加空格 ( 回车 ) 的情况和不加空格 ( 回车 ) 的情况冲突时, 优先加空格 ( 回车 )
  6. 每段后不加句号, 段中如果要使用句号, 请使用分号, 极特殊情况下 ( 如迁移的笔记 ) 可以使用英文句号
  7. 行尾如果符合上面所说需要加空格的情况, 也必须加空格
  8. 对于任何以上没有提到的情况, 如果看着不舒服, 可以加空格或回车, 但同样需要符合条件 6
  9. 禁止使用加粗, 因为当前的字体加粗不明显
  10. 在代码中应出现或已出现的变量名, 函数名, 类名, 导入的内容, 编程语言名等认为应当加注单引号格式的, 加注单引号格式

关于代码块

  1. 作为模板的代码必须有注释, 注释必须为行内注释, 注释符号前加两个空格, 符号后加一个空格, 其余与 "笔记内容" 相同
  2. 所有代码块必须标明语言, 缩进必须符合代码规范 ( 缩进为 4 个空格 ) , 禁止直接复制代码而不调整格式
  3. 复制的代码如果能找到来源, 加注来源, 如果是转载而又找不到来源, 不加注来源
  4. 如果将代码实现拆分成了几步, 已经实现的函数请用 "注释符号 + 函数声明 ( 对于 c++ ) / def 语句 ( 对于 python ) " 这样的格式代替, 缩进与原函数缩进相同
  5. 对于所有笔记中出现的代码, 请做好备份 ( 即使有略微更改也没有事 )

关于公式

  1. 公式代码不必追求整洁, 但公式外观必须整洁, 符号的要求与"笔记内容" 相同
  2. 公式中的空格宽度可以自行决定
  3. 公式中请避免出现中文, 但必须的中文也是允许的
  4. 换行请使用 \\ , 居中请使用 \begin{align} ... \end{align} 及 & 符号
  5. 含有复杂分数或其他导致公式比较庞大或过高的情况, 不要使用行内公式
  6. 严格禁止滥用公式
